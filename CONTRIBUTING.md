# Contributing

Thank you for helping improve this Linux security lab. Contributions should keep the repository clear, practical, and beginner-to-intermediate friendly.

## Filing Issues

Use issues to report documentation gaps, unclear instructions, broken commands, or ideas for future labs. Include:

- A short description of the problem or suggestion
- The file or lab affected
- Steps to reproduce the issue, when applicable
- Any relevant terminal output, with sensitive information removed

## Pull Requests

Pull requests should be focused and easy to review. Before opening a pull request:

- Keep changes limited to one topic
- Update related README files when behavior or structure changes
- Avoid adding unverified results or claims
- Remove private paths, usernames, tokens, IP addresses, and personal identifiers
- Explain what changed and why

## Coding Conventions

- Use clear file and directory names in lowercase with hyphens when possible
- Prefer portable shell commands for Linux examples
- Use Markdown headings instead of banner text
- Keep scripts readable and add comments only where they clarify intent
- Use safe defaults for examples, especially around networking and security tools
